========================================================
 REEL — your home movie server
========================================================

QUICK START (full details in README.txt):

  1. Install Node.js (LTS) from https://nodejs.org  — one time.
  2. Put your movie files in the  movies  folder.
       (Name them like:  The Big Trip (2023).mp4 )
       (MP4 / H.264 only — convert others with HandBrake.)
       You can also use the  Short Films ,  Documentaries , and
       Music Videos  folders — they show up as their own categories.
  3. Put TV shows in the  series  folder, one folder per show.
  4. START THE SERVER by double-clicking the launcher for your system:
       Windows:  start-windows.bat
       Mac:      start-mac.command   (right-click -> Open the 1st time)
       Linux:    start-linux.sh      ( ./start-linux.sh  in a terminal )
  5. It prints an address like  http://192.168.1.50:8080
     — open that on any phone, tablet, laptop, or TV on your WiFi.
     Leave the launcher window open while watching; close it to stop.

FOLDER LAYOUT
  start-windows.bat    <- double-click to start (Windows)
  start-mac.command    <- double-click to start (Mac)
  start-linux.sh       <- run to start (Linux)
  fix-sound-windows.bat  <- convert un-playable files (MKV/HEVC/AC3) -> MP4 (Windows)
  fix-sound-mac.command  <- convert un-playable files -> MP4 (Mac)
  fix-sound-linux.sh     <- convert un-playable files -> MP4 (Linux)
  server.js            <- the program that runs the server
  reel-prime.html      <- the app/page it serves  (keep this name)
  README.txt           <- full setup, profiles, PINs, admin, TMDB art
  movies\              <- DROP YOUR MOVIES HERE
  series\              <- DROP YOUR TV SHOWS HERE (one folder per show)
  Short Films\         <- short films (shown under "Short Films")
  Documentaries\       <- documentaries (shown under "Documentary")
  Music Videos\        <- music videos (shown under "Music Videos")
  data\                <- saved profiles, PINs (hashed), watch history

Files that appear automatically once you run it:
  data\profiles.json      <- profiles + watch history (created on first run)
  data\tmdb-cache.json    <- cached artwork lookups (only with a TMDB key)
